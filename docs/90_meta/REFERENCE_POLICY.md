# Política de Referências (Git-first) — appsicologa.cl

## Regra central
1) Git-first: antes de buscar fora, procurar no repo:
   - docs/00_maestro/ (canônico)
   - docs/01_playbook/
   - docs/02_branding/
   - docs/REFERENCIAS.md (registro)

2) Se usar fonte externa (API, dataset, doc oficial, guideline):
   - registrar/atualizar em docs/REFERENCIAS.md
   - registrar data de verificacao (YYYY-MM-DD)
   - registrar uso no produto (onde entra)
   - nao assumir endpoints/termos: confirmar e registrar

## Objetivo
- Evitar conhecimento solto fora do repo
- Pesquisa auditavel e repetivel
- Onboarding e execucao consistente
